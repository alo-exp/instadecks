# Phase 03 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution. Derived from RESEARCH.md §"Validation Architecture" and the 5-plan / 3-wave decomposition.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | `node --test` (Node 18+ built-in, zero-dep) |
| **Config file** | none (Phase 1/2 convention) |
| **Quick run command** | `node --test tests/<changed-file>.test.js` |
| **Full suite command** | `find tests -maxdepth 2 -name '*.test.js' -print0 \| xargs -0 node --test` (matches `ci.yml` Gate 6) |
| **Estimated runtime** | < 30 s full suite (no soffice) / < 90 s with `pptx-to-images.test.js` positive case on a soffice-available host |

---

## Sampling Rate

- **After every task commit:** Run quick command targeting the test file(s) the task touches (latency target < 30 s).
- **After every plan wave merge:** Run full suite command.
- **Before `/gsd-verify-work`:** Full suite green; `tools/validate-manifest.js` green; `tools/lint-paths.sh` green; `tools/assert-pptxgenjs-pin.js` green; license-checker green.
- **Max feedback latency:** < 30 s for changed-file quick run; < 90 s for full suite + Tier 1 SHA.
