   - 4-pass scan (MACRO / TYPOGRAPHY / DATA & OBJECTS / MICRO)
   - 4-tier severity (Critical 🔴 / Major 🟠 / Minor 🟡 / Nitpick ⚪) with calibration rule "when uncertain, choose the lower"
   - Finding grammar: `[Severity] | [Category] — [Location] — [Defect] — [Standard violated] — [Fix]`
   - §1 systemic / §2 design system / §3 per-slide / §4 maturity scoreboard / §5 top-10 fixes
   - Maturity rubric (verbatim, first-matching wins)
   - Anti-hallucination rules (re-expressed prose; rule #7 content-vs-design boundary preserved verbatim)
   - Large-deck chunking pattern (15-20 slides; chunk handoff carries §2 + prior counts)

The agent merges these into its findings array before calling `runReview`.
8. **R18 AI-tell detection — fuzzy side (LLM judgment):** explicit list:
   - Vague jargon ("synergize", "leverage", "unlock potential") — count per slide; flag if >2/slide.
   - Generic stock-photo vibes (people-pointing-at-screens, abstract handshakes).
   - AI-flavored phrasing ("In today's rapidly evolving landscape, …").
   - Inconsistent voice across speaker notes.
   - Title-case shouting.
   Each emitted as `category: 'style'` with `r18_ai_tell: true`, `genuine` per LLM judgment. Per P-08, agent flips `genuine: false` when the deck's design-rationale doc justifies the choice.
9. **Two-report architecture (D-06):**
   - `<deck>.review.md` — fixed template via `render-fixed.js`, deterministic, audit trail.
   - `<deck>.review.narrative.md` — LLM-authored, ≥200 words, MUST cite slide numbers + finding IDs (per finding text, not invented IDs). Audience: decision-maker who hasn't read §3 yet.
10. **--annotate gating (D-03):**
    - Triggers: explicit `--annotate` flag OR natural-language mention of "annotate" / "overlay" / "markup" / similar.
    - When triggered: pass `annotate: true` to `runReview`; output adds `<deck>.annotated.pptx` + `<deck>.annotated.pdf`.
    - Default = standalone (3 outputs only). Principle of least surprise.
11. **Severity-collapse boundary (P-01 restated):** "Producers (this skill, /content-review) emit FULL 4-tier severity. The 4→3 collapse to MAJOR/MINOR/POLISH happens ONLY at the /annotate adapter (Phase 2). Pre-collapsing here breaks /content-review and the future r18-only filter."
12. **Content-vs-design boundary:** "This skill flags visual / typographic / layout issues only. Argument structure, claim/evidence balance, narrative-arc problems are /content-review's territory (Phase 6). When you catch yourself writing 'the argument would be stronger if...' — DELETE the line."

```js
const test = require('node:test');
const assert = require('node:assert/strict');
const fs = require('node:fs');
const os = require('node:os');
const path = require('node:path');

const { runReview, _test_setRunAnnotate } = require('../skills/review/scripts/index');
const { detectAITells } = require('../skills/review/scripts/ai-tells');

test('happy path: ai-tells → runReview → outputs', async (t) => {
  // 1. Run code-side AI-tells against positive fixture.
  const aiTellFindings = await detectAITells(path.join(__dirname, 'fixtures', 'ai-tells-positive.pptx'));
  assert.ok(aiTellFindings.length >= 3, 'positive fixture should fire ≥3 AI-tell findings');

  // 2. Synthesize a minimal valid findings doc that wraps the ai-tells findings into the schema.
  const doc = {
    schema_version: '1.0',
    deck: 'tests/fixtures/ai-tells-positive.pptx',
    generated_at: '2026-04-28T00:00:00Z',
    slides: [/* 3 slides with the AI-tell findings distributed where slideNum is set; null-slideNum findings go in a §1 deck-systemic slot — exact wrapper is the test author's choice but must validate against findings-schema */],
  };

  // 3. Run runReview structured-handoff.
  const tmpDeck = ...; // touch a fake .pptx file; runReview standalone path doesn't open it
  const tmpOut = fs.mkdtempSync(path.join(os.tmpdir(), 'rvw-int-'));
  const r = await runReview({ deckPath: tmpDeck, findings: doc, mode: 'structured-handoff', outDir: tmpOut });

  assert.ok(fs.existsSync(r.jsonPath), 'sibling JSON written');
  assert.ok(fs.existsSync(r.mdPath), 'sibling MD written');
  assert.ok(typeof r.findingCounts === 'object', 'findingCounts present');
  assert.ok(typeof r.genuineCount === 'number', 'genuineCount present');
  // Narrative MD is authored by the AGENT post-runReview — assert it does NOT exist yet.
  assert.ok(!fs.existsSync(r.narrativePath), 'narrative MD authored by agent post-runReview');
});

test('--annotate path: runAnnotate stub fires', async (t) => {
  let annotateCalled = false;
  _test_setRunAnnotate(async ({ deckPath }) => {
    annotateCalled = true;
    return { pptxPath: deckPath.replace('.pptx', '.annotated.pptx'), pdfPath: deckPath.replace('.pptx', '.annotated.pdf') };
  });
  t.after(() => _test_setRunAnnotate(null));

  const r = await runReview({ deckPath: tmpDeck, findings: doc, mode: 'structured-handoff', annotate: true, outDir: tmpOut });
  assert.strictEqual(annotateCalled, true);
  assert.ok(r.annotatedPptx);
  assert.ok(r.annotatedPdf);
});
```
