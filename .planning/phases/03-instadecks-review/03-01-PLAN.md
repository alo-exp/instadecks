<interfaces>
**Script invocation contract** (consumed by Phase 3 `runReview` and Phase 4-6 callers):
```bash
${CLAUDE_PLUGIN_ROOT}/scripts/pptx-to-images.sh <input.pptx> <output_dir>
```
- Exit 0 = success; PDF + slide-N.jpg files present in `<output_dir>`.
- Exit 1 = invalid args / input not a file.
- Exit 2 = soffice failed twice.
- Exit 3 = soffice produced empty/missing PDF (post-call validation tripped).
- Exit 4 = pdftoppm failed twice.
- Exit 5 = pdftoppm produced no JPGs OR a JPG failed JPEG-magic / size check.

**Cleanup invariant:** `/tmp/lo-${SESSION_ID}-${PID}` is removed on EXIT/INT/TERM (RVW-11).

**Portable file-size idiom (Q-3):** `wc -c < "$file"` only — NEVER `stat -c%s` or `stat -f%z` (P-03).

**v8 reference body** (RESEARCH §"Code Examples — Example 1") — copy verbatim into `scripts/pptx-to-images.sh` with the exit-code map above; banner-comment header per Phase 2 §banner-comment pattern (path-lint-allow tokens not needed — no hardcoded /Users paths).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → shell script | User-supplied `<input.pptx>` and `<output_dir>` paths cross into bash; subject to shell-injection / path-traversal. |
| Shell script → soffice/pdftoppm | Shelled-out tools with known CVE history; isolated via per-PID user-instance dir. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | scripts/pptx-to-images.sh — shell injection via filename | mitigate | All variables double-quoted (`"$INPUT"`, `"$OUTDIR"`); `set -euo pipefail`; `umask 0077`. Future: add Node-side filename validation in callers (rejects `\n`, null bytes) before exec. |
| T-03-02 | Tampering | Path traversal via `<output_dir>` argument | mitigate | Caller (Phase 3 `runReview`) resolves `outDir` via `path.resolve` and asserts containment in `.planning/instadecks/<runId>/` before invoking the script. Phase 1 plans this guard into `runReview`. |
| T-03-03 | DoS | soffice hang / silent failure | mitigate | `timeout 60` per call; 1 retry then fail loud (exit 2/4); per-PID user-instance dir prevents cross-process wedging (D-08 inherited). |
| T-03-04 | DoS | soffice 0-byte PDF / corrupted output (P-02) | mitigate | Existence + size > 1024 + `%PDF` magic-byte checks (RVW-10) — exit 3 on any miss. |
| T-03-05 | Tampering | Stale `/tmp/lo-${SESSION_ID}-${PID}` accumulation | mitigate | EXIT/INT/TERM trap (RVW-11). |
| T-03-06 | Information Disclosure | soffice.stderr leaking absolute paths into log files | accept | Log files live in caller's `<outDir>` only; not committed; out-of-tree paths impossible because `outDir` is asserted within run-dir. |
</threat_model>
