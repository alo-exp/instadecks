From tools/validate-manifest.js (Node CLI pattern):
- Reads file, parses, asserts schema, exits 0/1 with structured error message.

SKILL.md frontmatter shape (all 4 existing skills):
```yaml
allowed-tools:
  - Bash(node:*)
  - Bash(soffice:*)
  - Bash(pdftoppm:*)   # review only
  - Bash(npm:*)        # content-review only
  - Bash(unzip:*)      # review, create, content-review
  - Bash(xmllint:*)    # create only
  - Read
  - Write
  - Glob               # content-review only
  - Grep               # content-review only
  - WebFetch           # create only
```

Pattern (FAIL):
- `Bash(*)` — unscoped wildcard
- `Bash` (no parens at all) — equivalent to wildcard
- `Bash(<cmd>)` without `:*` — too loose (e.g., `Bash(rm)` blocks rm but Bash(node) without `:*` blocks invocation, which we want to reject too)

Pattern (PASS):
- `Bash(<cmd>:*)` — scoped to a single binary with arg wildcard
- Non-Bash entries (Read/Write/Glob/Grep/WebFetch) — pass through
