**Filter helper (private, defined inside index.js):**
```js
function filterSlides(findings, slidesToReview) {
  if (slidesToReview == null || slidesToReview === 'all') return findings;
  if (!Array.isArray(slidesToReview)) {
    throw new Error(`runReview: slidesToReview must be null|'all'|int[] (got ${JSON.stringify(slidesToReview)})`);
  }
  for (const n of slidesToReview) {
    if (!Number.isInteger(n) || n < 1) {
      throw new Error(`runReview: slidesToReview entries must be positive integers (got ${JSON.stringify(n)})`);
    }
  }
  const keep = new Set(slidesToReview);
  return {
    ...findings,
    slides: (findings.slides || []).filter(s => keep.has(s.slideNum)),
  };
}
```

**Insertion point:** AFTER the `validate(findings)` call (so input integrity is preserved), BEFORE the run-dir resolve / writes. Filter is applied to a deep-clone-style spread; `findings` local var is reassigned so all downstream code (counts + JSON write + MD render) uses the filtered set. `findingCounts` and `genuineCount` recompute naturally from the filtered slides[] via the existing traversal at lines 35-50.

**P-01 invariant preserved:** the filter does NOT collapse severities or mutate finding shape — it only drops entire slide entries whose `slideNum` is not in the set. Severity collapse remains the annotate-adapter's job (CLAUDE.md locked invariant).

```markdown
### Scoped Review Mode (cycle 2+ in /instadecks:create auto-refine loop)

When the caller passes `slidesToReview: [3, 7, 9]`, scope the review to those
slide indices only. Skip the per-slide §3 entries for any slideNum not in the
list. The §1 systemic findings, §4 maturity scoreboard, and §5 top-10 fixes
sections still produce, but they reference only the in-scope slides.

Cycle 1 of the auto-refine loop ALWAYS passes null (full review). Cycle 2
when forced as the confirmation cycle (D-07) ALSO passes null. Cycle 2+
diff-only review passes the int[] of slides whose JPG SHA changed since the
prior cycle.

Empty list `[]` is valid — produces an empty findings.slides array (no
findings emitted); useful when no slides changed but the loop still wants a
sentinel review pass.

See also: `references/findings-triaged-schema.md` for the agent-augmented
output the auto-refine loop writes per cycle.
```
