# Project Knowledge — 2026-04

Concrete intelligence captured from the 10-phase v0.1.0 build.

### JSON contract first; annotate.js as consumer

**Category:** Key Decisions
**Phases:** 1, 2

The `/review → /annotate` JSON schema (`skills/review/references/findings-schema.md`) was locked in Phase 1 before any producer was written. `/annotate` shipped first as the smallest, most-locked component, validating the contract before reviewers were built. Every later phase triangulates on this schema; consumer-before-producer sequencing eliminated rework.

### 4-tier→3-tier severity collapse at /annotate adapter only

**Category:** Key Decisions
**Phases:** 1

Reviewers (`/review`, `/content-review`) emit Critical/Major/Minor/Nitpick in JSON. The collapse to MAJOR/MINOR/POLISH happens exclusively at the `/annotate` adapter when building SAMPLES for `annotate.js`. Two-stage taxonomy preserves reviewer fidelity while keeping annotate's geometry calibration unchanged. Source: `.planning/phases/01-instadecks-foundation/01-CONTEXT.md` D-decision.

### annotate.js verbatim + require-path patch + SHA pin

**Category:** Architecture Patterns
**Phases:** 2

`skills/annotate/scripts/annotate.js` is treated as a SHA-pinned binary asset. The only permitted edit is a documented one-line `require()` patch so pptxgenjs resolves from `${CLAUDE_PLUGIN_DATA}/node_modules`. Geometry math (polygon, charPts, miter-join, MAX_SIDE) is owned by direct unit tests added in Phase 8 (CONTEXT D-01 reversal). See CLAUDE.md locked invariants.

### soffice race fix via per-call -env:UserInstallation

**Category:** Known Gotchas
**Phases:** 3

Concurrent `soffice --convert-to pdf` invocations collide on the user-profile lockfile under `~/.config/libreoffice`. Fix: `scripts/pptx-to-images.sh` passes `-env:UserInstallation=file:///tmp/lo-$$-<rand>` per call, isolating profiles. Plan 03-01 also adds a no-op `timeout` shim on macOS dev hosts that lack GNU coreutils.

### Auto-refine convergence: genuine_findings==0 AND cycle≥2 + oscillation cycle N ⊆ cycle N-2

**Category:** Architecture Patterns
**Phases:** 5

The `/create` auto-refine loop converges when genuine_findings == 0 AND cycle ≥ 2 (cycle 1 with 0 findings forces one confirmation cycle). Oscillation: if cycle N's issue set ⊆ cycle N-2's, halt and surface the choice to the user. Soft cap at cycle 5; user-interrupt via `.planning/instadecks/<run-id>/.interrupt` flag. Locked in CLAUDE.md invariants.

### c8 100% as a hard CI gate (no exclusions)

**Category:** Key Decisions
**Phases:** 8

Phase 8 enforced `c8 --100 --check-coverage` (lines/branches/functions/statements) as Gate 6, with zero file exclusions. CONTEXT D-01 reversal placed `annotate.js` under standard test discipline. Plan 08-07 closed at 878 tests, 100% across all four metrics. The discipline of "no exclusions, no thresholds <100%" eliminated drift toward untested branches.

### bats local-only for bash; e2e gated by CI=true env

**Category:** Recurring Patterns
**Phases:** 8

CI runs `npm run test:bats` (Gate 5b/5c install + run); local hosts use `npm run test:bats:if-installed` to silently skip if bats absent. E2E tests live under `tests/e2e/` and are auto-skipped when `CI=true`; the `npm test` script prefixes `CI=true` so dev hosts with soffice locally also skip e2e (per CONTEXT D-08). FRESH-INSTALL.md remains the human v0.1.0 gate.

### Cookbook ≥3 variants per slide type + design-DNA rotation

**Category:** Architecture Patterns
**Phases:** 9

Phase 9 ships ≥3 cookbook variants per slide type and a design-DNA picker (palette × typography × motif rotation per deck) to defeat AI-tell uniformity. Validation: `tools/validate-cookbook.js` + cookbook-variant-coverage tests. Diversity is checked explicitly against prior runs.

### Polymorphic brief intake via lib/brief-normalizer.js

**Category:** Architecture Patterns
**Phases:** 9

Plans 09-04/05 introduce `skills/create/scripts/lib/brief-normalizer.js` to normalize heterogeneous input shapes (markdown, transcript, PDF text, freeform brief) into a uniform internal brief schema before render-deck.cjs synthesis. Invariant reversal validator (Plan 09-05) ensures the normalizer is idempotent on already-normalized input.

### cwd lock for parallel runCreate (.runCreate.lock + 30s soft-fail)

**Category:** Known Gotchas
**Phases:** 10

Parallel `runCreate` invocations sharing a working directory race on `${CLAUDE_PLUGIN_DATA}/state.json`. Plan 10-01 (HARD-02) introduces a `.runCreate.lock` file with a 30-second soft-fail: a second invocation that can't acquire the lock surfaces a clear error and exits non-zero rather than corrupting state. Lock auto-clears on graceful exit; stale locks are reaped by mtime check.
